# Настройка домашнего сервера

Я решил дать вторую жизнь старому ноутбуку и превратить его в домашний сервер, установив на него Ubuntu. Ниже — основные шаги настройки и доработки системы.

---

## Отключение реакции на закрытие крышки

По умолчанию ноутбук уходит в спящий режим при закрытии крышки, что неудобно для сервера. Это поведение можно изменить через конфигурацию systemd.

Открываем файл:

```
sudo nano /etc/systemd/logind.conf
```

Добавляем или изменяем параметры:

```
HandleLidSwitch=ignore
HandleLidSwitchExternalPower=ignore
HandleLidSwitchDocked=ignore
```

Описание параметров:
- `HandleLidSwitch` — действие при закрытии крышки на батарее
- `HandleLidSwitchExternalPower` — при работе от сети
- `HandleLidSwitchDocked` — при подключении к док-станции

Значение `ignore` означает, что система ничего не будет делать при закрытии крышки.

Применяем изменения:

```
sudo systemctl restart systemd-logind
```

## Отключение подсветки экрана при закрытии крышки

Чтобы не тратить лишнюю энергию, я настроил отключение экрана через bash-скрипт.

Сначала проверяем состояние крышки:

```
watch -n 1 'cat /proc/acpi/button/lid/LID0/state'
```

Затем создаем скрипт:

```
#!/bin/bash

if [ "$(cat /proc/acpi/button/lid/LID0/state | awk '{print $2}')" = "closed" ]; then
    xset dpms force off
fi
```

Сохраняем его:

```
/usr/local/bin/lid-screen-off.sh
```

Делаем исполняемым:

```
sudo chmod +x /usr/local/bin/lid-screen-off.sh
```

Добавляем в cron:

```
sudo crontab -e
```

И прописываем:

```
* * * * * /usr/local/bin/lid-screen-off.sh >> /var/log/lid-script.log 2>&1
```

Теперь экран будет автоматически отключаться при закрытии крышки.

## Назначение статического IP-адреса

Чтобы сервер всегда был доступен по одному адресу, задаем статический IP.

Сначала узнаем текущий интерфейс:

```
ip a
```

Редактируем конфигурацию netplan:

```
sudo nano /etc/netplan/01-network-manager-all.yaml
```

После изменений проверяем:

```
sudo netplan generate
```

Применяем:

```
sudo netplan apply
```

При ошибках можно использовать:

```
sudo netplan --debug apply
```

## Настройка файрволла (UFW)

Для базовой защиты сервера используется UFW (Uncomplicated Firewall).

### Установка (если не установлен)

```
sudo apt update
sudo apt install ufw
```

### Проверка статуса

```
sudo ufw status
```

### Базовые настройки

По умолчанию запрещаем все входящие соединения и разрешаем исходящие:

```
sudo ufw default deny incoming
sudo ufw default allow outgoing
```

### Разрешение необходимых портов

Разрешаем SSH (важно, чтобы не потерять доступ к серверу):

```
sudo ufw allow 22/tcp
```

Если используется веб-сервер:

```
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
```

### Включение файрволла

```
sudo ufw enable
```

Подтверждаем действие — после этого файрволл начнет работать.

### Просмотр правил

```
sudo ufw status verbose
```

### Удаление правила

```
sudo ufw delete allow 22/tcp
```

## Итог

В результате:
- ноутбук больше не засыпает при закрытии крышки
- экран отключается для экономии энергии
- сервер имеет статический IP
- настроен базовый файрволл для защиты

Теперь устройство готово к использованию в качестве домашнего сервера.
